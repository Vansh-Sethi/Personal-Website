Confusion Matrices Explained

2020-03-12

Explaining how these vital visual representations of machine learning models work.

A Data Scientists’ Dilemma

Big companies are getting involved with machine learning like never before. Especially with the necessity of data scientists steadily growing in big companies, data scientists bare a big job in the machine learning consulting space. One issue that arises from data scientists in big companies is the issue of communication with other branches in the company. Miscommunication and misinterpretation between data scientists and non-data scientists cost lots of money and time. The question becomes, how do data scientists communicate reports and metrics about their current projects to upper management? The answer used to be building a structure in the company with lots of confusion, however recently, the agreed-upon medium has become confusion matrices (it’s a bad name) which are visual representations of machine learning models.

Confusion Matrices — The General Explantation

To clearly understand how confusion matrices work, it is first important to understand the goal of any machine learning model. A machine learning model takes in an input and produces an output. Machine learning is where we make this process even better and teach the model how to give the desired output every time. There is a label to the input which is the true label and the predicted label which is the model’s output. **Confusion matrices are a visual representation of how accurate a model is for a test set. **A confusion matrix compares the true labels to the predicted labels of a model. It adds a value to a certain portion of a N x N grid, where N represents the number of labels in a test set. Here are the cases for the confusion matrix:

  • True Positive: The true label is the same as the predicted label, and the label represents a positive match in the data.

  • True Negative: The true label is the same as the predicted label, and the label represents a negative match in the data.

  • False Positive: The true label is different than the predicted label, and the true label is a negative match in the data whereas the predicted label is a positive match in the data.

  • False Negative: The true label is different than the predicted label, and the true label is a positive match in the data whereas the predicted label is a negative match in the data.

The way a confusion matrix is created and filled with values is part of the testing phase of the model. The model is run on a predefined amount of data points and is made to predict each and every one. For every data point and it’s prediction, one of the following cases will be filled and the grid corresponding to the case will go up by one. This happens until the whole test set is iterated through and the grid is filled up.

It’s important to know that the amount of cases for a confusion matrix grows as the number of labels goes up. The more labels, the more complex the confusion matrix will inevitably be.

A splash of colour!

Colour is very common in confusion matrices. Colour is used because of the need to communicate these models graphically and easier to upper management. The colour usually represents how much a certain grid was predicted. Usually, the more intense the colour, the higher the value of the grid. Using colours, it can be easier to tell how good a model is.

Pro tip: When there is a diagonal of intense colours, it usually means the model is good because those are the true positive grids.

Bridging the Gap of Machine Learning

Confusion matrices are making it much easier for anyone to understand how good a model performs. So the next time you see a confusion matrix, you can critique the model. Confusion matrices are fundamentally making it easier for upper management and other non-machine learning developers to understand how these models work, which saves lots of money and time. This is the wonderous world of confusion matrices.
